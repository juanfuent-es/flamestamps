Flame Tree Season Development proposal

Mobile-first campaign platform

Flame Stamps

Three production scopes for turning the visual prototype into a measurable campaign experience, a moderated archive, and a public participatory installation.

Overview

A campaign object that can grow from a personal stamp into a public artwork.

Flame Stamps begins as a mobile web experience for Flame Tree Season: visitors capture or upload an image, apply branded shapes and color treatments, generate a downloadable stamp, and keep or share the final output.

The proposal separates the work into three scopes. Each scope can be delivered independently, or combined as a progressive roadmap from launch platform to collective gallery to live installation.

Development scopes

Three ways to build the season

Proposal 1

Core Campaign Platform

A polished mobile-first platform where users create a branded stamp from camera capture or upload, save it to a personal album, and download or share the result.

  • Landing and mobile creation flow
  • Desktop fallback with QR handoff to mobile
  • Shape, filter and color treatment selection
  • Stamp generation, album, download and native share
  • Photo database by user/session ID and admin data tools
Estimated development
4 to 5 weeks

Proposal 2

Collective Gallery + Moderation

An extended platform where users can submit stamps to a shared campaign gallery, while the team reviews, approves, rejects and curates public submissions.

  • Everything in Core
  • Consent and submit-to-gallery flow
  • Pending, approved, rejected and private visibility states
  • Moderation dashboard with approve, reject and delete actions
  • Public gallery and approved content feed
Estimated development
6 to 8 weeks

Proposal 3

Living Tree Installation

A public display layer where approved user stamps become part of a growing digital flame tree shown on a screen, projection or venue display.

  • Everything in Core and Collective Gallery
  • Approved stamp feed for display mode
  • Basic crowdsourced tree for one device and resolution
  • Animation, fallback screen and technical handoff documentation
Estimated development
8 to 10 weeks

Scope comparison

From buildable core to full public participation layer.

The added technical layers clarify what is required to make the prototype production-ready. The split below keeps the Core focused, makes moderation mandatory only when content becomes public, and treats the tree as an installation layer that depends on curated data.

1. Core

Minimum production platform for users to create, save and share personal stamps.

Mandatory
Mobile capture/upload flow, album, basic animations, export/share, database with photos by user/session ID, desktop fallback with QR to mobile, and admin tools to inspect/delete data.
Pros
Fastest launch, measurable campaign participation, useful even without public publishing.
Optional
Full registration, CDN, extra UX mapping, and interface beautify beyond the current design system pass.

2. Collective Gallery

Public archive layer where personal stamps can be submitted, reviewed and published.

Mandatory
Everything in Core, consent flow, submission states, moderation dashboard, approve/reject/delete actions, public gallery and approved content feed.
Pros
Turns isolated outputs into campaign material, protects public display through review, and creates a reusable archive.
Optional
AI-assisted moderation, user accounts, advanced filtering, CRM hooks and CDN tuning.

3. Tree Installation

Display layer where approved stamps become part of a collective flame tree.

Mandatory
Everything in Core and Collective Gallery, approved stamp feed, basic crowdsourced tree designed and tested for one device/resolution, fallback state and operation handoff.
Pros
Creates the strongest public-facing moment and turns campaign participation into a living artwork.
Optional
Polished tree, responsive tree, WebGL/particles, live counters, location grouping and multi-screen deployment.

Additional layers

  • Registration: sign up, login, password management and verification. Approx. +10 days.
  • Polished crowdsourced tree: stronger motion, composition and visual finish. Approx. +7 days.
  • Responsive crowdsourced tree: multiple viewport/display layouts beyond one fixed resolution. Approx. +7 days.
  • AI moderation: semi-automated review/censorship support; still requires human moderation policy.
  • UX mapping: audit missing components, screens, edge states and admin flows before production.
  • CDN setup: improve load times for image-heavy campaign assets.
  • Interface beautify: refine the current UI while staying inside the Flame Stamps design system.

Recommended roadmap

Start with the individual ritual, then open it to the city.

The safest production path is progressive. First ship the core stamp experience, then add public participation and governance, and finally use the curated feed as material for an installation.

  1. Core Create, save and share personal Flame Stamps with backend data attached to each output.
  2. Collective Gallery Turn individual participation into a moderated public archive of approved campaign submissions.
  3. Tree Installation Transform curated stamps into a public participatory artwork that grows throughout the season.

Production notes

What must be defined before build

The prototype should be treated as a visual and conceptual reference. A production build needs clear decisions around data, consent, moderation, deployment and operational ownership.

Data and consent

If user images are stored or displayed publicly, the platform needs privacy copy, consent states, deletion paths and moderation before publication.

Campaign utility

The backend is a key value layer: every stamp becomes both a user-facing image and a measurable participation record for the campaign team.
