Flame Stamps

Un sistema editorial digital para convertir la temporada del flame tree en una experiencia visual, coleccionable y compartible.

Contenido de diseño

Editorial, cálido y móvil desde la primera pantalla

La identidad se apoya en lectura de alto contraste, serif variable, superficies oscuras y acentos de temporada. El sistema debe sentirse como una publicación digital con gesto botánico, no como una landing genérica de producto.

La composición base usa secciones de ancho completo, contenido contenido en columnas legibles y elementos visuales derivados de la marca: pétalos, recortes de árbol, sellos y el logotipo en sus variantes Flame Red, Light Yellow y Black.

Puntaje de contraste

18.61:1 Night #1C0F0A sobre Cream #FFFFF6. Base recomendada para cuerpo de texto y lectura larga.

Para botones de texto, Flame Red mantiene mejor contraste que Flame Orange. Orange queda reservado para énfasis, hover, selección y elementos visuales grandes.

Detalle técnico

Tokens mínimos para producción

Titulares

h1 usa --fs-h1 o --fs-hero, con Awesome Serif y variación alta: wght 540, hght 1000. La regla base viste el elemento, no una clase decorativa.

Subtitulares

h2 baja peso y aumenta aire: wght 320, line-height 1.16 y lectura balanceada. Debe abrir secciones editoriales, no competir con el título principal.

Cuerpo

p usa --fs-body, line-height 1.72 y --color-text-muted sobre Night. Para ledes, .lede sube tamaño y contraste sin abandonar el tono cálido.

Mapa de tokens

  • --color-primaryNight, superficie editorial principal.
  • --color-neutralCream, texto principal y fondo cálido claro.
  • --color-tertiaryFlame Red, interacción de texto con buen contraste.
  • --color-accentFlame Orange, energía visual y estados expresivos.
  • --fs-heroTítulo de primera pantalla, escalado con clamp().
  • --fs-bodyCuerpo editorial de lectura continua.

Reglas de guía de diseño

Cómo aplicar el sistema

Las reglas priorizan tokens, semántica HTML y aplicaciones reales de marca. Cuando haya que crear una nueva pantalla, primero se debe reutilizar color, ritmo, tipografía y assets existentes.

  • Vestir elementos base: definir h1, h2, p, section y a antes de inventar clases como .title o .body-copy.
  • Usar Night y Cream para lectura larga; usar Sage e Ink para secciones públicas o colectivas.
  • Aplicar Flame Red en CTAs con texto; reservar Flame Orange para acento, hover, selección o piezas gráficas.
  • Usar Amiri solamente en microcopy poético, etiquetas de temporada o captions breves.
  • Usar pétalos, árbol y logo desde `img/` antes de crear nuevas ilustraciones.
  • Evitar tarjetas genéricas anidadas, fondos grises, gradientes sintéticos y decoraciones que no vengan de la marca.
