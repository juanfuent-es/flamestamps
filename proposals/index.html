Dubai - Flame Tree Season Development proposal

Propuesta de sitio y plataforma participativa

Flame Stamps Campaign Platform

Una experiencia web mobile-first para crear estampas botanicas, convertirlas en participacion medible y escalar hacia archivo colectivo o instalacion publica.

Resumen editorial

Del sello personal a una capa publica de campana.

Flame Stamps parte de una accion simple: capturar o subir una imagen, vestirla con el sistema visual de Flame Tree Season y exportar una estampa lista para descargar o compartir.

La propuesta se organiza en tres niveles progresivos. El primero entrega la experiencia funcional y la base de datos; el segundo agrega publicacion, consentimiento y moderacion; el tercero transforma las estampas aprobadas en una pieza visual colectiva para pantalla o venue.

Tres alcances de desarrollo

Bloques progresivos, cada uno construido sobre el anterior.

01 - Core

Core Campaign Platform

Convierte el prototipo visual en una experiencia mobile-first donde cada usuario genera una estampa descargable y el sistema guarda participacion, imagen y metadata.

  • Landing, experiencia mobile y fallback desktop con QR.
  • Camara o carga de imagen, seleccion de forma, filtro y color.
  • Generacion de estampa, album personal, descarga y native share.
  • Base de datos con usuarios/sesiones, ID, fotos, stamps y metadata.
  • Dashboard basico de admin para revisar y eliminar data si es necesario.
Tiempo estimado
4 a 5 semanas

02 - Collective

Collective Gallery + Moderation

Agrega una capa publica y curatorial. Los usuarios pueden enviar su estampa a galeria, mientras el equipo aprueba, rechaza, elimina y prepara contenido para archivo.

  • Incluye todo el Core.
  • Flujo de envio a galeria con consentimiento de publicacion.
  • Estados pending, approved y rejected, con visibilidad publica o privada.
  • Galeria publica con feed aprobado y filtros por forma, color, filtro o fecha.
  • Moderacion con cola de revision, flagging y bloqueo basico por usuario/sesion.
Tiempo estimado
6 a 8 semanas

03 - Tree

Living Tree Installation

Usa las estampas aprobadas como componentes visuales de un arbol digital colectivo, pensado para una pantalla, proyeccion o display de venue.

  • Incluye Core y Collective Gallery.
  • Feed aprobado conectado a la visualizacion del arbol.
  • Arbol crowdsourced basico para un dispositivo o resolucion definida.
  • Animaciones basicas de interfaz y composicion con estampas aprobadas.
  • Refresco periodico, pantalla fallback y documentacion tecnica de operacion.
Tiempo estimado
8 a 10 semanas

Comparativa de valor

Que es mandatorio, que aporta valor y que puede quedar como capa opcional.

La lectura recomendada es incremental: Core crea el producto minimo de campana; Collective lo vuelve publicable y gobernable; Tree lo convierte en una instalacion visible. Saltar capas aumenta riesgo porque cada nivel depende de datos, consentimiento y moderacion del anterior.

Core

Entrega la experiencia individual completa y la infraestructura minima para que la campana tenga datos utiles.

Mandatorio
Mobile web, desktop QR, captura/carga, generacion de estampa, album, descarga/share, base de datos, storage y admin basico.
Pros
Menor riesgo, entrega rapida, base tecnica reutilizable y medicion real de participacion.
Opcional
Email/contact capture, QR o hashtag embebido, mayor polish visual y componentes extra de onboarding.

Collective

Vuelve la participacion publica sin publicar contenido automaticamente ni comprometer la seguridad de marca.

Mandatorio
Todo Core, consentimiento, estados de envio, galeria publica, feed aprobado y moderacion con approve/reject/delete.
Pros
Archivo curado, contenido reutilizable, control editorial y base directa para visualizaciones futuras.
Opcional
Filtros avanzados, exportaciones internas, ban/block mas granular, IA de premoderacion y dashboards extendidos.

Tree

Transforma el archivo aprobado en una pieza publica, pero requiere decisiones de resolucion, venue y operacion.

Mandatorio
Core y Collective, feed moderado, visualizacion del arbol, display target, animacion basica, fallback y handoff tecnico.
Pros
Mayor impacto publico, pieza participativa viva y narrativa clara entre usuario, campana e instalacion.
Opcional
Responsive/multi-resolucion, WebGL, particulas, crecimiento generativo, estados idle/active y agrupacion por metadata.

Capas adicionales

  • Beautify de interfaz: refinamiento visual siguiendo el sistema de diseno.
  • Registro de usuarios: sign up, login, password management y verificacion. +10 dias aprox.
  • Arbol mas pulido: composicion visual y motion design de mayor calidad. +7 dias aprox.
  • Arbol responsivo: soporte para multiples pantallas y resoluciones. +7 dias aprox.
  • IA de moderacion: censura o prefiltrado semi-automatizado con revision humana.
  • Mapeo UX: inventario de componentes, paginas y estados faltantes.
  • CDN: configuracion para acelerar imagenes, stamps y assets de campana.

Roadmap recomendado

Produccion por fases para reducir riesgo.

La ruta mas solida es entregar primero la experiencia individual, validar flujo y datos, y despues abrir capas publicas. La instalacion debe entrar cuando ya exista contenido moderado y una definicion clara de display.

  1. Core Campaign Platform Producto funcional para crear, guardar, descargar y medir Flame Stamps.
  2. Collective Gallery Capa publica con consentimiento, archivo curado y moderacion operable por el equipo.
  3. Living Tree Installation Visualizacion publica alimentada por stamps aprobados y preparada para un display definido.

Notas de produccion

El prototipo es referencia visual; la produccion necesita reglas operativas.

Si se almacenan imagenes de usuarios, deben existir privacidad, consentimiento y mecanismo de borrado. Si el contenido se muestra publicamente, la moderacion debe ocurrir antes de publicar.

Backend como valor

La base de datos convierte cada estampa en un objeto de campana: imagen, usuario o sesion, timestamp, seleccion visual, estado de publicacion y metadata util para analitica o curaduria.

Instalacion como scope separado

Venue, hardware, internet, pantallas, proyeccion, soporte en sitio, traduccion oficial, costos cloud y operacion extendida deben definirse fuera del alcance base.
